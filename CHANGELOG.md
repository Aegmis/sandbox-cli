## Unreleased

### Added

- `--paste` — mount `~/Desktop`, `~/Downloads` and `~/Pictures` read-only at
  their own host paths, so an image path pasted into an agent resolves inside the
  container. Pasting an image previously looked like it did nothing: the terminal
  inserts the file's absolute host path, and that path names nothing in a
  container whose only host mount is the project. Opt-in, because the agent can
  then read everything in those directories, not only the file that was pasted;
  read-only, because attaching an image never writes. Works on `run` and on every
  agent wrapper. It does not help an image copied as raw bits rather than as a
  file (a browser's "Copy Image", a screenshot straight to the clipboard) — there
  is no path involved and the container has no clipboard to read. See
  [docs/GUIDE.md](docs/GUIDE.md) for the narrower alternatives.
